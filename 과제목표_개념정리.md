# 과제 목표 개념 정리 (산출물 확인 위치 포함)

각 개념의 정의와 함께, 해당 개념이 본 과제 산출물의 어디에서 확인되는지 명시한다.

## 1. VSCode에서 Python 파일 생성·실행

개념:
- 파일 생성: 탐색기 새 파일 아이콘 클릭 → `.py` 확장자로 이름 지정
- 파일 실행: 터미널에서 `python3 파일명.py` 입력
- `.py`: 파이썬 소스 코드 파일 확장자

산출물 확인 위치:
- 파일 생성 결과: 탐색기 및 GitHub 저장소의 `main.py`
- 실행 결과: 프로그램 실행 결과 스크린샷 (`python3 main.py` 출력 화면)

## 2. 터미널에서 Python/Git 버전 확인·설정 점검

개념:
- `python3 --version` — 파이썬 버전 출력
- `git --version` — Git 버전 출력
- `git config --global user.name "이름"` — 커밋 작성자 이름 설정
- `git config --global user.email "이메일"` — 커밋 작성자 이메일 설정
- `git config --global --list` — 설정 목록 확인

산출물 확인 위치:
- 개발 환경 설정 스크린샷 (`python3 --version` → Python 3.12.13, `git config --global --list` → user.name, user.email, init.defaultbranch=main 출력 화면)

## 3. 파이썬 기초 문법

개념 및 `main.py` 내 확인 위치:

| 문법 | 정의 | main.py 내 위치 |

## 4. Git이 무엇이고 왜 필요한가

개념:

산출물 확인 위치:
- 변경 이력: `git log --oneline --graph` 스크린샷 (커밋 15개)
- 원격 관리: GitHub 저장소 `https://github.com/luvtaehee-prog/prompt-manager`

## 5. Git 명령 8개

개념 및 산출물 확인 위치:

| 명령 | 기능 | 확인 위치 |
|---|---|---|
| init | 로컬 폴더를 Git 저장소로 초기화 | 터미널 실행 화면 "빈 깃 저장소를 초기화했습니다" |
| add | 변경 파일을 커밋 대상으로 준비 | 터미널 `git add` 실행 화면 |
| commit | 준비된 변경을 이력으로 기록 | git log 스크린샷의 각 커밋(15개) |
| push | 로컬 커밋을 원격에 업로드 | git log 스크린샷의 origin/main 라벨 위치, 터미널 "main -> main" 출력 |
| pull | 원격 변경을 로컬로 병합 | git log 스크린샷의 병합 커밋 bc9fa76, 터미널 "이미 업데이트 상태입니다"/"Fast-forward" |
| clone | 원격 저장소 복제 | 터미널 "'Hello-World'에 복제합니다" 출력 화면 |
| checkout | 브랜치 생성·전환 | git log 스크린샷의 feature/prompt-list 라벨, 터미널 "새로 만든 'feature/prompt-list' 브랜치로 전환합니다" |
| merge | 브랜치 병합 | git log 스크린샷의 병합 커밋 bc9fa76, 3d9b756 |

확인 방법 구분:
- git log 스크린샷 1장으로 확인: commit, checkout, merge, push(간접), pull(간접)
- 터미널 실행 화면 스크린샷 필요: init, add, clone

## 6. 브랜치 생성·병합

개념:
- 브랜치: 독립된 작업 라인
- 생성·전환: `git checkout -b 브랜치명`
- 병합 절차:
  1. `git checkout main` — 병합 받을 브랜치로 이동
  2. `git merge --no-ff 브랜치명` — 대상 브랜치 병합
- `--no-ff`: 병합 커밋을 생성하여 그래프에 분기·병합 흔적을 남김

산출물 확인 위치 (`git log --oneline --graph` 스크린샷):
- 브랜치 생성: `7729630 (feature/prompt-list)` 라벨
- 병합 커밋: `3d9b756 merge: 프롬프트 목록 기능 병합`
- 분기·병합 흔적: `|\`, `|/` 선

## 7. GitHub에 코드 업로드·관리

개념:
- 업로드 절차:
  1. GitHub에서 원격 저장소 생성
  2. `git remote add origin URL` — 로컬과 원격 연결
  3. `git add .` → `git commit -m "메시지"` → `git push`
- 관리: `git log --oneline --graph`(이력 확인), `git pull`(원격 동기화)

산출물 확인 위치:
- 업로드 결과: GitHub 저장소 페이지 (파일 3개 .gitignore, README.md, main.py, 15 Commits 표시)
- 이력 관리: git log 스크린샷
